The difference between HTML and CSS

If you're looking at a website, it will consist of both HTML and CSS. If you were to think of HTML and CSS as characters, HTML would be the builder and CSS would be the artist. The artist cannot exist/do their job, without the builder laying the foundations before them. HTML is what gives your website all it's structure and the elements inside that structure. Such as your navigation bar, your headings, your paragraphs, your pictures and anything else you might wish to put on a website. CSS is going to determine what all of this looks like. Whether it makes your headings have a font of 'comic-sans', your background is bright pink, or your main picture wildly huge or teency tiny. By the power of HTML and CSS, you have the power to make your own beautifully customisable website!

... so what is JavaScript and how does that fit in? If JavaScript were a character it would be the sorceress. A badass witchy womxn making the website interactive and doing magical things. It basically does everything else that a website can do. It can even change your HTML and CSS content!

Control Flow and Loops

Explain control flow and loops using an example process from everyday life, for example 'waking up' or 'brushing your teeth'. (But not those ones).

Control flow is the order in which the computer executes a script.

Looping in programming languages is a feature which facilitates the execution of a set of instructions/functions repeatedly while some conditions execute to true.

Control flow is the order in which all the stuff in our code is executed. Imagine if control flow in coding was like doing the laundry. First, we need to fill up our laundry basket with dirty clothing, put the contents into the washing machine, add laundry powder, turn the washing machine on and then finally once it's all clean, hang it out to dry! It wouldn't make sense to hang our dirty laundry out to dry and then put the washing machine on without anything in it! This is why control flow is important as it takes all the instructions needed to complete a certain task, and puts them all in a flowing order.

Looping, in this analogy, could be the repetitive chugging of the washing machine as it completes a certain amount of chugs before it knows to move onto the next phase of the washing cycle. This loop is a repetitive task that needs to be completed a certain number of times before the task is considered done (or in the coding world - when a function has been executed to a set amount of times).

What is the DOM?

DOM stands for 'Document Object Model'. And no, it's not the coding overlord in bondage gear... The DOM is actually the special glue that ties HTML, CSS and JavaScript together. It's the programming interface for HTML and it is the object-oriented representation of a webpage.

When we open up our Dev Tools (in chrome say), this is the visual represntation of the DOM. And yeah! It looks pretty similar to the HTML code that's written behind it. The DOM can be modified with a scripting language such as JavaScript (which the browser reads) and inside the DOM is where all the stuff happens. What kind of stuff you say? Well it could be stuff like watching and waiting for a mouseclick or a keyboard stroke and then the reaction that happens is going to be visible in the DOM.

Accessing data from arrays and objects

Explain the difference between accessing data from arrays and objects.

Both arrays and objects store data. But they do so in different ways that can be useful for different purposes. In arrays, the order of the data matters and is assigned a numerical value (in [square brackets]), whereas in objects, we'll call upon the characteristic in that object that we're interested in. Each object has a key and a value.
